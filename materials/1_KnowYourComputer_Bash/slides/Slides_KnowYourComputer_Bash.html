File naming conventions

  • No space characters or punctuation other than hyphens (-), underscores (_), and periods (.) should be used in filenames
  • Choose a capitalization strategy and observe it consistently throughout your project
  • Filenames that contain multiple logical words can avoid the use of spaces and still maintain legibility by using underscores, hyphens, or camel case, e.g., “index_places.txt”, “index-places.txt”, or “indexPlaces.txt”. Choose a convention and use it consistently
  • If you keep multiple versions or drafts of files in the same directory, include a date stamp in the filename. We recommend a year-month-date format, with a four-digit year and two-digit months and dates, e.g., “2016-07-30_index-places.txt”.
  • Filenames should have a meaningful filename extension, like .xml, .txt, .docx

Exercise

  1. Navigate to the location where you want to download EnExDi2020 materials and back to home using Bash. To move around use cd, to know where you are use pwd, and to lists the files and folders use ls. Don't forget that auto-completion (the 'tab' key) can help you.
  2. Write down in your notes (not in the terminal) the absolute path to that location, because we will use it a lot during the week. Remember that absolute paths starts with "/", from /home for Mac and Linux (for Windows, keep reading). The drawing you prepared before should help here.
    For example: /home/espadini/Documents.
    Attention, for Windows the absolute path should start with /cygdrive/c/Users/ and then your login name. For example: /cygdrive/c/Users/elena/Documents.
  3. Test this absolute path. Wherever you are in the terminal, type cd and the absolute path.
    For example: cd /home/espadini/Documents.

3. On the importance of readme

'Readme' files generally describe the content of the directory or give information about how to use the files next to it.

'Readme' files are .txt (plain text) or .md (markdown) files. Plain text is text without any additional formatting or encoding. Markdown is a lightweight markup language, a way to style elements in your texts, such as titles, links, lists, codes and other elements (see a markdown cheat sheet); some programs are able to render markdown.

Use Sublime to open .txt and .md files (even if Markdown is rendered only partially; a plugin is available if needed).
